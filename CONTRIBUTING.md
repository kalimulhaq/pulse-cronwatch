# Contributing

Thank you for considering contributing to `pulse-cronwatch`. Bug reports and pull requests are welcome.

## Reporting bugs

Open an issue on [GitHub](https://github.com/kalimulhaq/pulse-cronwatch/issues) with:

- The Laravel, Pulse, PHP and `pulse-cronwatch` versions you are running.
- The smallest reproduction you can produce.
- The behaviour you expected vs. what actually happened.

## Sending a pull request

1. Fork the repository and create a topic branch.
2. Run the test suite locally: `composer install && vendor/bin/phpunit`.
3. Add tests for any behaviour change.
4. Follow [PSR-12](https://www.php-fig.org/psr/psr-12/) for code style.
5. Keep commits focused — one logical change per commit.
6. Open a pull request describing the *why* as well as the *what*.

## Security

If you discover a security vulnerability, please email `kalim.dir@gmail.com` directly rather than filing a public issue.
